Installation

gaudi uses Docker to start all applications in a specific container. It builds Docker files and specific configuration files from different templates.
All templates are listed in the templates/ folder, one for each application type.

Requirements

gaudi requires Docker to run.

OSX / Windows: Using Vagrant

The Cethy/vagrant-gaudi repository describes how to install gaudi with Vagrant.

A puppet module is available to install gaudi.

Options

To start gaudi, run sudo gaudi in the current folder.

List of options:

--config="" Specify the location of the configuration file
--debug Display some useful information
--no-cache Do not use docker's cache when building containers (builds will be slower)
--quiet Do not display build & pull output
--empty-cmd Use `/bin/bash` for `CMD` on specified container (separated by comma, can be `all`. eg: `---empty-cmd=app,db`). Useful for debugging a container
rebuild Force all containers to rebuild (useful when you change your configuration)
stop Stop all applications
check Check if all applications are running
run binaryName [arguments] Run a specific binary
