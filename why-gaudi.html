Why gaudi ?

Provisioning and orchestration can be painful for a developer with no system administration knowledge.

With orchestrator like Chef or Puppet, you have to find the good module and learn how to make them work together.
Gaudi is build on top of Docker. It can run each component of your architecture in a container.
Each container are isolated and can communicate when they are linked.

Gaudi allows to configure each component individually, depending of they type.
In this way you can configure where to redirect PHP request in a Nginx component, using replication with MySQL container, etc...
